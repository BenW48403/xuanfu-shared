## 总则 1 · 素材结构模型（M1/M4 必读)
交互场景 = **1 张 base 背景(全闭基准态)** + **N 个独立叠加精灵**,每精灵带状态序列。
- **M1 出图法**:先出 base;每个精灵态用**图生图 + 区域原位重绘(inpaint)**在 base 同坐标同光照下改局部,叠回严丝合缝。**不为状态组合各出整张背景**(背景不变,变的是叠加精灵 → 加法不乘法)。
- 录屏中"餐桌区随进度叠加开箱/开屏风/开柜"等快照已实证此模型:背景恒定,差异全在精灵态。
- M4 按游戏状态独立切每个精灵显隐/换态。

## 总则 2 · 解锁前置门控（D-07 核心,视频看不出)`[B]`
**每个密码/拼图谜题有「前置线索集」,未集齐则答案正确也不响应,杜绝盲猜。** 逐谜见 D-07。

## 总则 3 · 线索卡 / 线索层 / 查看层 `[B]`
- **道具卡**:道具的名片;道具 = **能收进道具栏**的物品。
- **线索卡(clue)**:不可拾取、属背景的线索物(石碑/书本/笔记)。M1 出 **1 图 = 原貌**(`clue_<id>`,文字歪扭/斑驳、竖排、从右到左)。
- **线索层**:点击线索卡弹出,呈现原貌,比精灵层多一个「查看」按钮。
- **查看层**:点「查看」→ **由 M4 程序直出**(黑底 + **白色情节文字** + **蓝色线索文字**),**非 M1 素材**。拜堂的地支数字书内容全为线索(蓝);到原创阶段情节(白)/线索(蓝)并存,靠颜色分。

### 道具总清单 + 状态机
剪刀钳·蜡烛·火柴·浆糊·冥钞·红布·秤杆·钥匙A(麻将盒)·钥匙B(花轿)·剪刀·蝴蝶饰品1·蝴蝶饰品2·弓箭·玉手镯 = 普通道具。
- **红包**:不可拾取 → 装冥钞后可给(给小孩)。
- **寿衣**:不可交互,属开箱后箱内背景。
- **地支数字书**:线索卡(非道具,见总则3)。
- **钥匙A** 汇 = 灯笼廊↔月亮门锁门;**玉手镯** 持有即解卧室门(不交给娘子)。
