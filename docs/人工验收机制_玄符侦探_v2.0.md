**一、核心原则（v2.0更新）**

> v2.0新增原则：已通过人工审批的素材，其生成提示词必须同步归档至D-09.5素材总清单和D-10美术风格指南。锁定的提示词是全系列视觉/音频风格的唯一标准，后续同类素材必须以此为基础变体。

- 任何文档、图像、音频、代码，未获人工审批通过前，下一阶段Agent不得启动

- 已审批素材的生成提示词立即归档，状态变更为LOCKED，作为全系列风格锚点

- 当前阶段（Demo）：通过文档审批和图片集合审批代替飞书群通知

- 未来阶段（Demo通过后）：接入飞书平台，逐张图片在群内推送审批

**二、素材逐张审批微流程**

**2.1 标准流程（当前Demo阶段）**

|          |              |                                                                                                              |
|----------|--------------|--------------------------------------------------------------------------------------------------------------|
| **步骤** | **执行方**   | **操作**                                                                                                     |
| **1**    | M1/M2/M3     | 生成素材，将文件名、生成提示词、素材图/音频文件一并整理，提交给P1                                            |
| **2**    | P1 总策划    | 汇总该批次所有素材，按D-09.5素材总清单顺序排列，一次性提交给审批人（你），并说明本批次是第几批、共多少个文件 |
| **3**    | 审批人（你） | 逐张/逐个查看素材，对每一个回复「通过」或描述修改意见。通过后，该素材进入下一步                              |
| **4**    | P1 总策划    | 收到「通过」后，立即在D-09.5对应行填入「锁定提示词」，将状态从IN REVIEW变更为LOCKED                          |
| **5**    | P1 总策划    | 同步将锁定提示词归档至D-10美术风格指南对应章节（图像素材）或D-12音频设计文档（音频素材）                     |
| **6**    | P1 总策划    | 确认该批次所有素材LOCKED后，解锁依赖该批次的下一批次（WAITING→PENDING），通知对应Agent开始生产               |

**2.2 打回返工流程**

|          |              |                                                                             |
|----------|--------------|-----------------------------------------------------------------------------|
| **步骤** | **执行方**   | **操作**                                                                    |
| **1**    | 审批人（你） | 说明具体问题：哪里不对、期望是什么（风格/内容/细节）                        |
| **2**    | P1 总策划    | 将审批意见转达给负责Agent，在D-09.5将该素材状态标记为REJECTED，附注打回原因 |
| **3**    | M1/M2/M3     | 根据审批意见修改后重新提交，P1重新整理提交审批                              |
| **4**    | 审批人（你） | 复核修改版，通过后流程回到2.1步骤4                                          |

> 关键规则：同一素材最多打回3次。第3次仍不通过时，P1需重新审查上游文档（D-06角色设定或D-05场景氛围）是否存在描述不清的问题，可能需要修改上游文档后重新生成。

**三、提示词归档机制**

**3.1 归档位置**

|                |                                                 |                                                       |
|----------------|-------------------------------------------------|-------------------------------------------------------|
| **素材类型**   | **归档位置**                                    | **归档内容**                                          |
| **场景背景图** | D-09.5「锁定提示词」列 + D-10美术风格指南第三章 | 完整的Midjourney/SD提示词，含风格词、构图词、技术参数 |
| **道具图标**   | D-09.5「锁定提示词」列 + D-10美术风格指南第四章 | 提示词含道具材质、光影方向、透明背景处理方式          |
| **NPC精灵图**  | D-09.5「锁定提示词」列 + D-10美术风格指南第五章 | 基础版本提示词；变体版本只记录「在基础版本上的差异」  |
| **UI元素**     | D-09.5「锁定提示词」列 + D-11 UI视觉规范        | 设计说明+参考锚点（参考哪个已锁定的场景图色调）       |
| **音频文件**   | D-09.5「锁定提示词」列 + D-12音频设计文档       | Suno/Udio完整提示词，含风格词、乐器、时长、情绪       |

**3.2 跨章节复用规则**

- 第一章通过审批的素材提示词，自动成为第二章及后续章节的「风格基准」

- 第二章生成同类素材时，M1必须以第一章锁定提示词为基础，只修改「场景描述词」和「内容词」，不修改「风格词」和「技术参数」

- 如需刻意改变风格（如第三章进入不同地点），需要向P1提出「风格变更申请」，经人工审批后才能修改基准提示词

**五、OpenClaw规则预定义（Demo通过后实施）**

> 以下规则在Demo验证全部通过后，将被转化为OpenClaw工作流配置文件。当前阶段作为规则预定义文档存档。

**5.1 飞书群审批触发规则**

|                  |               |                                                                                                                                     |
|------------------|---------------|-------------------------------------------------------------------------------------------------------------------------------------|
| **触发条件**     | **触发Agent** | **飞书群消息格式**                                                                                                                  |
| **素材生成完成** | M1/M2/M3      | 「【审批请求】R-XX 第X批 第X/N张」 文件名：xxx.png 类型：场景背景图 生成提示词：\[完整提示词\] \[图片附件\] 请审批：✅通过 / ❌打回 |
| **审批人回复✅** | P1 总策划     | 自动触发：①D-09.5状态更新为LOCKED ②提示词归档至对应文档 ③若本批次全部LOCKED，通知下一批次Agent开始生产                              |
| **审批人回复❌** | P1 总策划     | 自动触发：①D-09.5状态更新为REJECTED ②提取审批人的修改意见 ③转发给负责Agent并要求修改                                                |

**5.2 状态回写规则**

- 每次状态变更（PENDING→IN
  REVIEW→LOCKED/REJECTED）必须同步写入D-09.5素材总清单

- LOCKED状态变更时，同步将提示词写入D-10/D-11/D-12对应章节

- 所有状态变更记录保留变更时间戳，作为审计记录

- P1每完成一个批次的状态回写后，在工作群发送「第X批已全部LOCKED，共X件素材，第X批生产已启动」

**5.3 OpenClaw Agent配置要求（待实现）**

- P1 Agent需要有飞书消息推送权限和D-09.5文档写入权限

- M1/M2/M3
  Agent在完成素材生成后，自动触发「提交审批」工作流，无需人工介入

- 审批人的飞书回复需要被P1 Agent实时监听，触发后续自动化动作

- D-09.5素材总清单建议使用飞书多维表格实现，支持状态字段和附件字段，替代docx格式

> 飞书多维表格方案：将D-09.5的表格迁移到飞书多维表格后，Agent可以直接通过API读写状态字段和提示词字段，比操作docx文件更稳定高效。Demo阶段用docx，正式运营阶段迁移到飞书多维表格。
