**D-03 文件命名规范文档**

所有Agent生成资源前必须阅读本文档，命名不规范的文件M4将拒绝集成

文档版本：v1.0 \| 负责Agent：P1 总策划 \| 输出给：M1、M2、M3、M4

**一、命名总原则**

- 全部使用英文小写字母、数字、下划线（\_），禁止使用空格、中文、特殊字符

- 格式：类型前缀 \_ 章节编号 \_ 描述 \_ 变体（可选）. 扩展名

- 章节编号固定两位：第一章 = ch01，第二章 = ch02，以此类推

- 描述词使用英文，多个单词用下划线连接，不超过3个词

- 所有文件名不超过50个字符

> 核心公式：{类型前缀}\_{章节}\_{描述}\_{变体}.{扩展名}
> 例：bg_ch01_hall_mid.png

**九、禁止命名方式**

> 以下命名方式M4将直接拒绝集成，M1/M2/M3必须严格避免：

|                                     |                                            |
|-------------------------------------|--------------------------------------------|
| **错误示例**                        | **问题说明**                               |
| 背景图-大厅.png                     | 含中文，操作系统路径兼容性问题             |
| BG Hall Mid.png                     | 含空格和大写，代码引用时容易出错           |
| image001.png                        | 无语义，M4无法识别资源用途                 |
| bg_hall_final_FINAL_v3_use_this.png | 版本信息不应在文件名中体现，用版本控制管理 |
| npc_ghost.png（不含章节）           | 跨章节资源命名冲突风险                     |
