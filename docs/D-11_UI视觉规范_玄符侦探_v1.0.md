**一、设计原则**

> D-11是全系列UI的视觉标准。通用UI（R-12a审批）通过后，本文档锁定，后续章节直接复用。

- 克制优先：UI元素不抢夺场景视觉注意力，使用半透明叠加而非实色遮挡

- 中式线描风格：边框使用仿古线描纹样，避免现代扁平化圆角矩形

- 功能明确：每个UI元素在0.5秒内可被新玩家理解其功能

- 暗色优先：UI背景使用深色（墨绿/赭石系），与明亮场景形成层次

**三、通用UI元素规范**

**3.1 底部道具栏**

- 位置：屏幕底部，高度80px，宽度覆盖全屏（1920px）

- 背景：深墨绿半透明（80%不透明度），上边框旧金色线描纹

- 道具槽：8格，每格120×68px，间距6px；空槽显示槽位底纹

- 选中状态：槽位边框变为烛火黄，亮度提升，有轻微发光效果

- 主线道具（T-12铜牌）：右下角显示「主线」红色标记，不可拖出

**3.2 右上角提示「?」按钮**

- 尺寸：64×64px，圆形，位于屏幕右上角距边缘24px

- 默认状态：深墨绿底+旧金色「?」字符+旧金色圆形边框

- 激活状态（进入提示模式）：边框变为烛火黄，底色加亮

- 当前场景所有热点已完成时：按钮半透明（60%）暗示无热点可显示

**3.3 场景内「?」热点标记**

- 尺寸：40×40px，圆形，带脉冲动画（2秒循环轻微缩放）

- 样式：月白底色+深墨绿「?」字符，四周烛火黄光晕

- 位置：紧贴可交互元素右上角或右侧，不遮挡主体

- 进入提示模式时出现，退出提示模式或对应任务完成后消失

**3.4 底部提示菜单栏**

- 触发方式：点击场景内某个「?」热点后，从底部滑入覆盖道具栏

- 高度：80px（与道具栏同高），背景深墨绿实色

- 图标组从左到右：灯泡（已解锁/120px宽）/ 影片（需广告/120px宽）/
  锁头（未解锁/120px宽）/ 红叉（关闭/80px宽）

- 灯泡图标：金色灯泡，选中时蓝色背景高亮；影片图标：胶片框样式；锁头图标：灰色锁头；红叉：红色×

- 关闭后道具栏重新显示

**3.5「◁」退出提示模式按钮**

- 尺寸：48×48px，三角形向左箭头

- 位置：屏幕左侧中央，距左边缘16px

- 仅在提示模式激活时显示

- 样式：半透明深色背景+月白箭头，与场景不争抢视觉

**3.6 NPC对话框**

- 样式：仿古卷轴，上下卷轴纹，背景深墨绿半透明（85%）

- 位置：屏幕下方1/3处，宽度800px，居中

- NPC名称显示于卷轴顶部，字体思源宋体Bold/24px，朱砂红色

- 文字逐字显示动画（20字/秒），点击任意处跳过动画

**四、谜题UI通用规范**

> 谜题UI每套单独审批（R-12b），但以下规范适用于所有谜题UI，M2设计时必须遵守。

- 进入谜题界面时，全屏覆盖谜题背景（1920×1080px），场景不可见

- 左侧固定显示「◁」退出按钮（与提示模式退出按钮同样式）

- 完成状态必须有明显的视觉反馈：烛火黄光晕+成功音效（由M3提供）

- 错误状态：界面轻微震动（CSS animation）+朱砂红边框闪烁

- 所有谜题界面背景统一使用深墨绿+旧木纹底纹，保持与场景风格一致
