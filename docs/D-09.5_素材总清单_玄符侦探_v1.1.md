**一、v1.1修正说明**

> v1.1重大修正：原v1.0严重低估了场景变体图数量。原因是只考虑了「门开/关」类变体，未考虑「道具取走后容器变空」和「NPC消失后场景变化」两类变体。根据参考游戏截图分析和项目负责人确认，所有含道具的容器在道具被取走后，背景图必须更新为「已开启/已取空」状态，以给玩家明确的进度提示，减少无效操作。

**二、状态标记说明**

|           |                                                      |
|-----------|------------------------------------------------------|
| **状态**  | **含义**                                             |
| PENDING   | 等待生成——素材尚未由负责Agent生成                    |
| IN REVIEW | 审批中——已生成，已提交人工审批                       |
| LOCKED    | 已锁定——人工审批通过，提示词已归档，作为后续风格基准 |
| REJECTED  | 已打回——不通过，负责Agent修改后重新提交              |
| WAITING   | 等待依赖——依赖其他素材审批通过后才能开始生成         |
| **新增**  | v1.1新增的素材行（绿色标记）                         |

> LOCKED状态素材的提示词是全系列风格标准。后续章节同类素材必须以LOCKED提示词为基础进行变体，不得重新生成风格。
